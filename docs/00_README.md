# 项目最终执行版文档包说明

项目名称：**《基于 STM32F103C8T6 的端云协同 AI 多模态安全巡检与设备健康监测系统》**

报告封面名称：**《基于 STM32F103C8T6 与 ESP32-CAM 的端云协同 AI 安全巡检与设备健康监测系统》**

上报表名称：**STM32 端云协同 AI 多模态安全巡检与设备健康监测系统**

答辩标题名称：**基于 STM32 的端云协同 AI 多模态安全巡检系统**

---

## 1. 项目一句话定位

系统以 STM32F103C8T6 为实时安全主控，融合烟雾、火焰、人体红外、门磁、温湿度、振动等多模态传感数据，构建本地风险评分与异常分级状态机；异常发生时，本地立即完成声光报警、继电器保护、风扇/水泵联动，同时调度 ESP32-CAM 进行异常现场抓拍，并将图片、传感器快照与风险等级上传至 Web/手机端或服务器。服务器侧可调用云端 AI 进行图像复核、异常原因解释和处理建议生成，形成端侧实时控制、边缘视觉取证、云端智能分析相结合的端云协同 AI 安全巡检系统。

---

## 2. 核心约束

| 约束 | 执行要求 |
|---|---|
| 主控 | STM32F103C8T6 是系统唯一主控制器 |
| 协处理 | ESP32-CAM 负责抓拍、联网、Web 页面、图片上传、AI 请求中转，不直接控制关键执行器 |
| 本地闭环 | 不接 ESP32-CAM、不联网、不使用 Web/MQTT/AI 时，STM32 本地系统仍必须能完整演示 |
| 关键执行器 | 蜂鸣器、RGB LED、继电器、水泵、风扇等只能由 STM32 控制 |
| AI 定位 | 云端 AI 参与异常复核、图像解释、处理建议生成和事件归类，但不覆盖 STM32 本地安全判断 |
| 远程管理 | Web Dashboard、手机端页面、服务器数据上报是智能化远程管理层，不是安全闭环前置条件 |
| 难度来源 | 多模态感知融合、状态机、风险评分、主从通信协议、云端 AI 集成、优雅降级和系统工程可靠性 |
| 不做内容 | 不做复杂机械臂、小车、PCB 打样、从零训练模型、FOTA、真实高压强电演示 |

## 3. 四层系统架构

```text
第四层 ─ 云端 AI 智能解释层
  图像复核 · 异常类别识别 · 多模态信息解释 · 处理建议生成 · JSON 格式返回
  AI 结果用于展示和日志，不覆盖 STM32 本地判断
       ▲
       │ HTTP / AI API
第三层 ─ 服务器 / Web / 手机端 远程管理层
  事件数据接收 · 图片文件接收 · 当前状态展示 · 历史事件查询
  手机端浏览 · 可选 MQTT/HTTP · 局域网优先
       ▲
       │ Wi-Fi / HTTP / MQTT
第二层 ─ ESP32-CAM 边缘视觉协处理层
  异常抓拍 · 本地图片缓存 · Web 预览 · 网络连接 · 状态回传
  摄像头心跳 · 抓拍失败降级
       ▲
       │ UART 主从协议
第一层 ─ STM32 本地实时控制层
  多传感器采集 · 风险评分 · 状态机 · OLED · 按键
  蜂鸣器/RGB LED · 继电器 · 水泵/风扇 · 本地事件日志
```

## 4. 功能层级

| 层级 | 含义 | 依赖关系 |
|---|---|---|
| 基础安全闭环层 | STM32 本地闭环，可独立演示 | 不依赖 ESP32-CAM、网络、Web、AI |
| 边缘视觉取证层 | ESP32-CAM 抓拍、UART 协议、心跳、降级 | 依赖基础安全闭环层稳定 |
| 端云协同与远程可视化层 | Web Dashboard、手机端、服务器、MQTT/HTTP | 依赖基础安全闭环层，部分依赖边缘视觉取证层 |
| 云端 AI 智能解释层 | AI 图像复核、异常解释、处理建议 | 依赖边缘视觉取证层和服务器层 |

## 7. 最小成功标准

只要完成以下闭环，项目就具备可提交基础：

```text
烟雾/火焰/PIR/门磁任一触发
→ STM32 本地采集
→ 风险评分
→ 状态机进入预警/报警/危险
→ OLED 显示状态和风险等级
→ 蜂鸣器/RGB LED 反馈
→ 风扇/继电器/水泵至少一个执行器动作
→ 按键可布防、撤防、静音、复位
```

---

## 8. A 类冲刺成功标准

在最小成功基础上，进一步完成以下内容可显著提升选题评级：

```text
→ ESP32-CAM 异常抓拍成功（SNAP_OK 可回显）
→ Web Dashboard 可在手机浏览器查看当前状态和抓拍图片
→ 服务器接收并存储至少一条事件和图片
→ 云端 AI 返回异常解释 JSON 并展示在 Web 页面
→ 断网/AI 失败时本地系统继续工作，Web 显示"AI 分析不可用"
```
