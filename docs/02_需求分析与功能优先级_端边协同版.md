# 02 需求分析与功能优先级（OPi5 主线版）

## 2.1 功能需求按四层组织

### 2.1.1 第一层：本地感知与安全控制

| 需求 | 优先级 | 说明 |
|---|---|---|
| GPIO 输入采样 | 必须 | PIR/火焰/MQ-2 真实输入 |
| 执行器默认安全态 | 必须 | 蜂鸣器、RGB、MOS 负载上电默认 OFF |
| `risk_score` 计算 | 必须 | 0–10 风险评分，由本地规则计算 |
| `safety_fsm` | 必须 | NORMAL/WARN/VERIFY/ALARM/FAULT |
| 离线降级 | 必须 | AI/Flask 不可达时仍能本地报警或记录 |
| systemd 常驻 | 必须 | `opi5-safetyd.service` 崩溃自动重启 |

### 2.1.2 第二层：视觉采集与视频流

| 需求 | 优先级 | 说明 |
|---|---|---|
| USB 摄像头接入 | 必须 | OPi5 USB 口接入免驱摄像头 |
| device-agent 视频流 | 必须 | MJPEG 流 `/api/video/stream`，mock 回退 |
| device-agent 快照 | 必须 | `/api/video/snapshot.jpg`，mock 需标注 |
| Flask 视频代理 | 推荐 | `/api/video/stream?device_id=` 转发 |
| Pan 舵机扫描 | 推荐 | 单轴 MG90 三点扫描 |

### 2.1.3 第三层：事件存储与可视化

| 需求 | 优先级 | 说明 |
|---|---|---|
| Flask 事件 API | 必须 | `/api/events` 兼容旧字段 |
| 遥测批量上报 | 必须 | `/api/telemetry/batch`（30s） |
| AI observation 上报 | 必须 | `/api/ai/observations`（30s） |
| React edge-console | 必须 | Dashboard 多页面可视化 |
| 通知设置 | 推荐 | email 通知 + 配置持久化 |
| mock/seed 数据 | 推荐 | 硬件不可用时 mock 数据验证 |

### 2.1.4 第四层：本地 AI 推理与解释

| 需求 | 优先级 | 说明 |
|---|---|---|
| mock AI 服务 | 必须 | 先打通 API，不阻塞联调 |
| Qwen3-VL 2B | 必须 | 当前主线 VLM 模型 |
| AI observation 生成 | 必须 | device-agent 每 30s 调用 AI |
| YOLO 目标检测 | 加分 | person/fire/smoke 等 |
| 人脸识别 | 加分 | known/unknown |

## 2.2 非功能需求

| 类别 | 目标 | 验收方式 |
|---|---|---|
| 响应时间 | 人机尺度 100ms–500ms 级本地联动；AI 可异步 | 测试记录端到端耗时 |
| 局域网可用性 | PC/WSL、OPi5 能互通 | ping/ssh/curl 记录 |
| 离线降级 | AI/Flask 不可达时状态机不崩溃 | 断网测试 |
| 可调试性 | 每个模块能独立命令行验证 | tests 目录留证据 |
| 可演示性 | 3–5 分钟内展示完整链路 | docs/14 脚本 |

## 2.3 安全性需求

- 不使用 220V 负载。
- 舵机、MOS 负载独立供电并共地。
- AI 不直接控制执行器；`control_allowed=false`。
- MOS 栅极下拉，控制程序启动前负载默认 OFF。
- 所有真实配置使用本地文件，不提交密钥、IP、WiFi 密码。

## 2.5 优先级划分

### 必须完成

- OPi5 safetyd 本地安全闭环（GPIO 传感器 → 状态机 → 执行器）
- OPi5 device-agent（心跳/遥测/视频/AI observation）
- Flask backend API（事件/遥测/AI/通知）
- React edge-console（Dashboard 可视化）
- Qwen3-VL AI 服务
- mock 传感器/视频数据验证
- email 通知

### 推荐完成

- Pan 舵机巡检扫描
- 真 RKNN demo
- 通知设置持久化

### 加分项

- 人脸识别
- rknn-llm 解释
- MPU6050 设备健康监测

### 明确不做

- 从零训练大模型
- 机械臂/小车/PCB 打样
- 公网云平台强依赖
- 220V 强电控制
