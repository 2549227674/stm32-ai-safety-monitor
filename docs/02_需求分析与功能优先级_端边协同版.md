# 02 需求分析与功能优先级（端边协同版）

## 2.1 功能需求按四层组织

### 2.1.1 第一层：本地感知与安全控制

| 需求 | 优先级 | 说明 |
|---|---|---|
| GPIO 输入采样 | 必须 | 至少接入门磁/PIR/火焰/按键中的一路真实输入 |
| 执行器默认安全态 | 必须 | 蜂鸣器、RGB、MOS 负载上电默认 OFF |
| `risk_score` 计算 | 必须 | 0–10 风险评分，由本地规则计算 |
| `safety_fsm` | 必须 | NORMAL/WARN/VERIFY/ALARM/FAULT |
| 离线降级 | 必须 | OPi5/Flask 不可达时仍能本地报警或记录 |
| systemd 常驻 | 推荐 | 控制进程崩溃自动重启 |

### 2.1.2 第二层：视觉采集与云台巡检

| 需求 | 优先级 | 说明 |
|---|---|---|
| USB 摄像头识别 | 必须 | i.MX6ULL 出现 `/dev/video<TODO:VERIFY>` |
| V4L2 静帧抓拍 | 必须 | 事件触发或定时抓拍 |
| 图片上传 | 推荐 | 上传到 Flask `/api/images` 或发送到 OPi5 |
| MG90 云台扫描 | 推荐 | pan/tilt 简单三点扫描 |
| 目标跟踪 | 加分 | 根据 AI 置信度停留到目标方向 |

### 2.1.3 第三层：事件存储与可视化

| 需求 | 优先级 | 说明 |
|---|---|---|
| 保留 `/api/events` | 必须 | 兼容旧事件 JSON |
| 扩展 AI 字段 | 必须 | 支持 `vision/ai_result/image_url/latency_ms` |
| Dashboard 展示 | 必须 | 显示 state、risk、传感器、执行器、AI 摘要 |
| 图片展示 | 推荐 | 显示最新抓拍图或链接 |
| 最终部署到 OPi5 | 推荐 | 减少 PC 依赖，利用 NVMe 存储 |

### 2.1.4 第四层：本地 AI 推理与解释

| 需求 | 优先级 | 说明 |
|---|---|---|
| mock AI 服务 | 必须 | 先打通 API，不阻塞端边联调 |
| RKNN 目标检测 | 推荐 | person/fire/smoke 等 |
| 人脸识别 | 加分 | known/unknown，比对本地向量库 |
| 本地 LLM 解释 | 加分 | rknn-llm 生成事件摘要，不控制执行器 |

## 2.2 非功能需求

| 类别 | 目标 | 验收方式 |
|---|---|---|
| 响应时间 | 人机尺度 100ms–500ms 级本地联动；AI 可异步 | Task07 记录端到端耗时 |
| 局域网可用性 | PC/WSL、i.MX6ULL、OPi5 能互通 | ping/ssh/curl 记录 |
| 离线降级 | OPi5/Flask 不可达时状态机不崩溃 | 断网测试 |
| 可调试性 | 每个模块能独立命令行验证 | tests 目录留证据 |
| 可演示性 | 3–5 分钟内展示完整链路 | docs/13 脚本 |

## 2.3 安全性需求

- 不使用 220V 负载。
- 舵机、风扇、水泵、继电器负载独立供电并共地。
- AI 不直接控制执行器。
- MOS 栅极下拉，控制程序启动前负载默认 OFF。
- 所有真实配置使用本地文件，不提交密钥、IP、WiFi 密码。

## 2.4 优先级划分

### 必须完成

- Task01–07。
- Flask 兼容扩展。
- i.MX6ULL 运行至少一个本地硬件控制程序。
- OPi5 AI 服务返回结构化 JSON。

### 推荐完成

- PCA9685 + MG90 云台演示。
- 真 RKNN demo。
- Dashboard 图片展示。

### 加分项

- 人脸识别。
- rknn-llm 解释。
- PREEMPT_RT 或 watchdog 细化。
- MPU6050 设备健康监测。

### 明确不做

- 从零训练大模型。
- 机械臂/小车/PCB 打样。
- 公网云平台强依赖。
- 220V 强电控制。
