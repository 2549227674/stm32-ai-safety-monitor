# 08 OPi5 视觉采集与视频流方案

## 8.1 当前目标

OPi5 USB 摄像头 → `opi5-device-agent` → Flask 视频代理 → React edge-console。

## 8.2 device-agent 视频接口

`opi5-device-agent` 提供以下视频端点：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/video/stream` | MJPEG 视频流（`multipart/x-mixed-replace`） |
| GET | `/api/video/snapshot.jpg` | 单帧 JPEG 快照 |

- 摄像头可用时：真实 USB 摄像头采集
- 摄像头不可用时：自动回退 mock 模式
- mock 模式必须在画面上标注 "MOCK" 或在响应中标明 `mock=true`
- 设备状态通过 `/api/status` 可查询摄像头是否在线

## 8.3 Flask 视频代理

Flask 后端提供代理端点，转发 device-agent 的视频流：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/video/stream?device_id=` | 代理 MJPEG 流 |
| GET | `/api/video/snapshot.jpg?device_id=` | 代理快照 |

代理地址解析优先级：

1. `OPI5_DEVICE_AGENT_URL` 环境变量
2. heartbeat `agent_url` 字段
3. heartbeat `ip` + `agent_port` 推断

ip 或 agent_url 为 unknown/null/空时，返回 503 JSON（`{"ok": false, "error": "device agent not available"}`），不请求 `http://unknown:8090`。

> seed 脚本（`scripts/seed_frontend_mock_data.py`）只演示 telemetry/AI/通知 mock 数据，不承诺视频流可用。视频流需要实际启动 opi5-device-agent：`VIDEO_MOCK=1 python3 edge/opi5-device-agent/app.py`

## 8.4 前端展示

React edge-console 视频区域行为：

| camera_status | video_mode | 前端显示 | 说明 |
|---|---|---|---|
| `online` | `real` | REAL CAMERA · LIVE | 真实 USB 摄像头 |
| `mock` | `mock` | MOCK VIDEO · 模拟中 | 真实摄像头失败后 fallback 或显式 mock |
| `offline` | `offline` | NO SIGNAL | 设备离线或代理不可达 |
| `unknown` | `unknown` | LIVE（不阻断） | 状态未知，允许视频流尝试 |

## 8.5 AI 观察

device-agent 每 30s 生成一次 AI observation：

1. 抓取当前帧快照
2. 汇总最近 30s 遥测数据
3. 调用 OPi5 AI 服务（`/api/infer/vision`）
4. 上报 `/api/ai/observations`

| 字段 | 说明 |
|---|---|
| `summary` | AI 自然语言摘要 |
| `risk_hint` | 0–10 风险提示 |
| `full_text` | AI 完整输出 |
| `objects[]` | 检测到的目标 |
| `run_metrics` | 推理耗时等指标 |
| `model` | 使用的模型名 |

> AI observation 超过 90s 未更新时，前端不应展示为"最新"。

## 8.6 不再使用

以下视觉方案属于历史阶段，不在当前 OPi5 主线中：

| 方案 | 历史阶段 | 说明 |
|---|---|---|
| i.MX V4L2 抓拍 | 第二版 i.MX 控制 | 已迁移到 device-agent |
| pan/tilt 二维云台扫描 | 第二版 PCA9685 | 当前仅单轴 Pan |
| PCA9685 舵机控制 | 第二版 i.MX I2C | 模块问题，已弃用 |
