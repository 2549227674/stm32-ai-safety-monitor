# 08 OPi5 视觉采集与视频流方案

## 8.1 当前目标

OPi5 USB 摄像头 → `opi5-device-agent` → Flask 视频代理 → React edge-console。

## 8.2 device-agent 视频接口

`opi5-device-agent` 提供以下视频端点：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/video/stream` | MJPEG 视频流（`multipart/x-mixed-replace`） |
| GET | `/api/video/snapshot.jpg` | 单帧 JPEG 快照 |

- 摄像头可用时：真实 USB 摄像头采集
- 摄像头不可用时：自动回退 mock 模式
- mock 模式必须在画面上标注 "MOCK" 或在响应中标明 `mock=true`
- 设备状态通过 `/api/status` 可查询摄像头是否在线

## 8.3 Flask 视频代理

Flask 后端提供代理端点，转发 device-agent 的视频流：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/video/stream?device_id=` | 代理 MJPEG 流 |
| GET | `/api/video/snapshot.jpg?device_id=` | 代理快照 |

当前代理逻辑：Flask 通过 `OPI5_DEVICE_AGENT_URL` 环境变量定位 device-agent 地址，转发对应端点。设备不在线时返回 503。

> 下一轮代码修复将支持从 heartbeat 的 `agent_url` 字段自动发现 device-agent 地址，无需手动配置环境变量。

## 8.4 前端展示

React edge-console 视频区域行为：

| 状态 | 展示 |
|---|---|
| 真实视频 | MJPEG 流实时显示 |
| 设备离线 | "无信号" 占位状态 |
| mock 模式 | 视频区域标注 "MOCK" |

## 8.5 AI 观察

device-agent 每 30s 生成一次 AI observation：

1. 抓取当前帧快照
2. 汇总最近 30s 遥测数据
3. 调用 OPi5 AI 服务（`/api/infer/vision`）
4. 上报 `/api/ai/observations`

| 字段 | 说明 |
|---|---|
| `summary` | AI 自然语言摘要 |
| `risk_hint` | 0–10 风险提示 |
| `full_text` | AI 完整输出 |
| `objects[]` | 检测到的目标 |
| `run_metrics` | 推理耗时等指标 |
| `model` | 使用的模型名 |

> AI observation 超过 90s 未更新时，前端不应展示为"最新"。

## 8.6 不再使用

以下视觉方案属于历史阶段，不在当前 OPi5 主线中：

| 方案 | 历史阶段 | 说明 |
|---|---|---|
| i.MX V4L2 抓拍 | 第二版 i.MX 控制 | 已迁移到 device-agent |
| pan/tilt 二维云台扫描 | 第二版 PCA9685 | 当前仅单轴 Pan |
| PCA9685 舵机控制 | 第二版 i.MX I2C | 模块问题，已弃用 |
