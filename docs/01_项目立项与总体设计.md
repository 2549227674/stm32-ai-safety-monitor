## 1.2 项目背景

实验室设备柜、储物柜、机房设备箱等场景可能存在烟雾、火焰、高温、非法打开、人员靠近、设备被搬动或异常震动等多种安全风险。传统单传感器报警器只能完成"触发蜂鸣器"的简单响应，缺乏多源融合判断、异常分级控制、执行器联动保护、异常现场取证、远程可视化管理和智能化异常分析能力。

本项目构建以 STM32F103C8T6 为核心的端云协同 AI 安全巡检系统。STM32 作为本地实时安全主控，完成多模态传感器采集、风险评分、状态机控制和执行器联动；ESP32-CAM 作为边缘视觉协处理器，在异常发生时完成现场抓拍取证；抓拍图片与传感器快照通过 Wi-Fi 上传至 Web/手机端和服务器，服务器可调用云端 AI 进行图像复核、异常类别识别和处理建议生成。系统实现"端侧实时控制 → 边缘视觉取证 → 远程可视化管理 → 云端智能分析"的四层端云协同架构，同时保证即使网络、AI、摄像头全部失效，STM32 本地安全闭环仍能独立运行。

## 1.5 核心创新点

| 创新点 | 体现方式 |
|---|---|
| 多模态感知融合 | 烟雾、火焰、PIR、门磁、温湿度、震动六类传感数据融合风险评分 |
| 本地实时闭环 | 所有安全判断和执行器控制由 STM32 完成，不依赖外部系统 |
| 边缘视觉取证 | 异常发生时 STM32 调度 ESP32-CAM 自动抓拍，保留现场证据 |
| Web/手机端可视化 | 手机浏览器即可查看当前状态、历史事件和抓拍图片 |
| 数据上传与事件存储 | 传感器快照、事件记录、图片路径上传至服务器持久化 |
| 云端 AI 异常解释 | 云端 AI 进行图像复核、异常分类、风险评估和处理建议生成 |
| 优雅降级 | ESP32-CAM 离线、网络失败、AI 超时均不影响本地安全闭环 |
| 主从协同通信 | UART 帧协议、ACK/NACK、心跳、超时重传、错误码 |
| 设备健康监测 | MPU6050 轻量震动分析，RMS/峰峰值/方差判断设备异常 |
| 系统工程可靠性 | 供电安全、水电隔离、故障标志、降级策略、防误报机制 |

## 1.6 与普通项目的区别

| 对比对象 | 普通做法 | 本项目做法 |
|---|---|---|
| 普通烟雾报警器 | MQ-2 超阈值 → 蜂鸣器响 | 多传感器融合 → 风险评分 → 状态机分级 → 执行器联动 → 抓拍取证 → AI 解释 |
| 普通传感器实验 | 单传感器单输出 | 六类传感器、多级状态机、多执行器、通信协议、降级策略 |
| 普通 ESP32-CAM 项目 | ESP32-CAM 为主控看画面 | ESP32-CAM 只被 STM32 调度，不控制执行器 |
| 普通物联网作品 | 云平台为核心，断网不可用 | 本地闭环为核心，云端 AI 为智能增强 |
| 普通传感器上云 | 传感器数据直传云平台 | 本地先完成安全判断和保护动作，再上传数据和请求 AI |

## 1.8 端云协同数据流图

```text
正常监控：
  传感器 → STM32 采集 → 风险评分 → 状态机(NORMAL) → OLED 绿灯

异常触发：
  传感器异常 → STM32 风险评分升高 → 状态机(ALARM/DANGER)
    ├→ 本地执行：蜂鸣器+RGB+风扇/继电器/水泵 【立即，<1秒】
    ├→ OLED 显示异常类型和风险等级
    ├→ UART 发送 SNAP_REQ → ESP32-CAM 抓拍 → 返回 SNAP_OK/FAIL
    │    ├→ 抓拍成功：图片缓存/Web 预览
    │    │    ├→ HTTP 上传事件 JSON + 图片 → 服务器存储
    │    │    │    └→ 服务器调用云端 AI → 返回异常解释 JSON
    │    │    │         └→ AI 结果展示在 Web/手机端
    │    │    └→ 手机浏览器查看当前状态 + 抓拍图片 + AI 解释
    │    └→ 抓拍失败：OLED 显示 SNAP_FAIL，本地报警继续
    └→ ESP32-CAM 离线：OLED 显示 CAM_OFFLINE，本地报警继续

断网/AI 失效：
  STM32 本地安全闭环继续运行
  Web/手机端显示"AI 分析不可用"或"网络离线"
  所有执行器动作不受影响
```

## 1.9 为什么这是 AI 项目而不是普通传感器报警器

本项目在传统传感器报警基础上增加了三个 AI 相关维度：

**维度一：多模态感知融合（端侧智能）**。不是单传感器单阈值判断，而是六类传感器数据融合的风险评分引擎，通过加权、连续确认、多源交叉验证实现智能化异常分级，这本身就是一种轻量级端侧智能。

**维度二：视觉取证与边缘协同（边缘智能）**。异常发生时自动触发摄像头抓拍，将"纯传感数据"升级为"传感数据 + 视觉证据"的多模态信息包，为后续 AI 分析提供视觉输入。

**维度三：云端 AI 异常解释（云端智能）**。将传感器快照和抓拍图片组织为结构化 JSON，调用云端视觉模型或大模型 API 进行图像复核、异常类别识别、场景描述和处理建议生成。AI 结果用于增强人类决策，而非替代本地安全判断。

这三个层次共同构成"端侧感知 → 边缘取证 → 云端智能"的端云协同 AI 架构，区别于普通 C 类传感器报警项目。

## 1.10 为什么 AI 不直接控制水泵

在安全关键系统中，AI 推理存在延迟、网络存在中断、模型存在误判。如果将执行器控制权交给 AI 或远程端：一旦网络中断或 AI 超时，系统将无法完成应急保护动作。因此本项目采用"本地判断优先、AI 辅助解释"的架构：STM32 本地传感器在毫秒级完成风险判断和执行器控制；AI 在秒级返回解释和建议。这是安全关键系统的工程最佳实践，也是本项目的设计亮点之一。

## 1.11 一句话项目介绍

本项目以 STM32F103C8T6 为实时安全主控，融合六类多模态传感器构建本地风险评分与状态机，联动声光报警和执行器保护；异常时调度 ESP32-CAM 抓拍取证，通过 Web/手机端实现远程可视化管理，并接入云端 AI 进行图像复核与异常解释，形成端侧实时控制、边缘视觉取证、远程管理和云端智能分析相结合的端云协同 AI 安全巡检系统。
