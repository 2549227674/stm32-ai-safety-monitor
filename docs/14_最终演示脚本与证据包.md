# 14 最终演示脚本与证据包

## 14.1 演示前 5 分钟检查

### 全无线网络检查

```bash
# i.MX 上执行
/etc/init.d/S45wifi-client status
ip addr show wlan0
env | grep -i proxy || echo "NO_PROXY"
curl -sS --connect-timeout 5 http://<OPI5_WIFI_IP>:8080/health
curl -sS --connect-timeout 5 http://<PC_WIFI_IP>:5000/api/status/latest || true
```

```bash
# OPi5 上执行
nmcli dev
lsmod | grep -E '8188|r8188'
env | grep -i proxy || echo "NO_PROXY"
curl -sS --connect-timeout 5 http://127.0.0.1:8080/health
```

```powershell
# Windows PowerShell
ipconfig
ping <IMX_WIFI_IP>
ping <OPI5_WIFI_IP>
curl.exe --connect-timeout 5 http://<OPI5_WIFI_IP>:8080/health
curl.exe --connect-timeout 5 http://<PC_WIFI_IP>:5000/api/status/latest
```

### Flask 启动

```bash
cd server/backend
python3 app.py --host 0.0.0.0
```

Windows 防火墙需放行 5000 端口。

## 14.2 全无线网络检查

三台设备均连接同一手机热点：

```text
手机热点 (10.96.98.0/24)
├── PC / Windows — Flask Dashboard
├── OPi5 — AI Service
└── i.MX6ULL — 本地主控
```

IP 由热点 DHCP 分配，演示前需确认。

## 14.3 i.MX 本地状态机演示

- PIR / 火焰 / MQ-2 传感器触发本地 ALARM
- 蜂鸣器 + RGB 状态灯
- OLED 显示状态
- 继电器 / 水泵动作
- 断网时本地报警仍可工作

## 14.4 OPi5 AI 解释演示

- i.MX 抓拍 → POST OPi5 AI → 返回 risk_hint / summary / objects
- `control_allowed=false`
- AI 不直接控制执行器

## 14.5 Dashboard 展示演示

- Flask Dashboard 显示事件、传感器、执行器、AI 结果
- 实时更新

## 14.6 故障降级演示

- 关闭 OPi5 AI 服务 → i.MX 生成 offline fallback AI
- 断开网络 → i.MX spool 事件，恢复后 flush
- 本地传感器仍触发 ALARM

## 14.7 回退方案

| 优先级 | 方案 | 说明 |
|---|---|---|
| 1（当前） | 全无线热点 | 三台设备同一热点 |
| 2 | Windows portproxy | i.MX 有线 → PC → OPi5 WiFi |
| 3 | 全有线 | i.MX → OPi5 有线 10.0.1.120 |
