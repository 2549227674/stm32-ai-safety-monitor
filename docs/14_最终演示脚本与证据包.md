# 14 最终演示脚本与证据包（OPi5 版）

## 14.1 演示前 5 分钟检查

### Flask 启动（PC）

```bash
cd server/backend
python3 app.py --host 0.0.0.0
```

Windows 防火墙需放行 5000 端口。

### OPi5 服务检查

```bash
# OPi5 上执行
systemctl status opi5-safetyd
systemctl status opi5-ai-qwen3vl
systemctl status opi5-device-agent

# 健康检查
curl -sS http://127.0.0.1:8080/health          # AI 服务
curl -sS http://127.0.0.1:8090/api/status       # device-agent
curl -sS http://127.0.0.1:5000/health           # Flask backend
```

### Dashboard 打开

浏览器打开 `http://<PC_IP>:5000` 或 React edge-console 地址。

## 14.2 Mock 数据演示（设备不可用时）

当硬件设备不可用时，使用 mock 数据演示完整功能：

```bash
# 注入 mock 数据
python scripts/seed_frontend_mock_data.py
```

展示内容：
- Dashboard 设备列表（mock 设备）
- 遥测时序图（mock 传感器数据）
- AI observation（mock 推理结果）
- 通知设置与测试邮件
- 视频流区域标注 "MOCK"

## 14.3 真实设备演示（设备可用时）

| 步骤 | 内容 | 说明 |
|---|---|---|
| 1 | 展示 OPi5 硬件 | 传感器（PIR/火焰/MQ-2）、执行器（buzzer/RGB/MOS）、OLED、摄像头 |
| 2 | 触发传感器 | PIR → VERIFY，火焰/MQ-2 → ALARM，buzzer/RGB 动作 |
| 3 | 展示 AI 推理 | device-agent 调用 Qwen3-VL，返回 `risk_hint`/`summary`，`control_allowed=false` |
| 4 | 展示 Dashboard | Flask API 数据 → React edge-console 实时展示 |
| 5 | 展示视频流 | MJPEG 流或快照 |

## 14.5 故障降级演示

| 场景 | 操作 | 期望 |
|---|---|---|
| AI 服务停止 | `systemctl stop opi5-ai-qwen3vl` | safetyd 仍依据本地传感器 ALARM |
| video 离线 | 不启动 device-agent | Dashboard 显示无信号 |
| Flask 离线 | 停止 Flask | device-agent 事件进入 spool |

## 14.6 回退方案

| 优先级 | 方案 | 说明 |
|---|---|---|
| 1 | OPi5 本地演示 | 不依赖 PC，OPi5 直接运行所有服务 |
| 2 | PC Flask + OPi5 | 局域网连接 |
| 3 | 全 mock | seed 数据演示完整功能 |
