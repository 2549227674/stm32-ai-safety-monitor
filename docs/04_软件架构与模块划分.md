# 04 软件架构与模块划分

## 4.1 架构选择

推荐采用：**裸机前后台 + SysTick 定时调度 + 串口中断接收 + 状态机**。

不强制使用 RTOS，原因：STM32F103C8T6 资源有限，裸机足够完成课程项目；两周周期内裸机更容易调试和稳定；状态机、定时任务、协议解析已经能体现系统工程复杂度。

STM32 端开发方式已经定稿为：**STM32CubeMX + Keil MDK-ARM + HAL 库 + ST-Link**。后续源码统一使用 HAL API，不使用也不混用 STM32F10x 标准外设库。当前 P0 阶段不引入 FreeRTOS。

CubeMX 生成文件中的自定义代码必须写在 `USER CODE BEGIN` / `USER CODE END` 区域内，避免重新生成工程时丢失。`.c/.h` 源码注释建议使用英文，降低 Keil 中文注释显示乱码的风险；README 和 docs 文档可继续使用中文。

## 4.4 模块职责表

| 模块 | 职责 |
|---|---|
| sensor_manager | 统一采集所有传感器并生成 SensorData |
| risk_engine | 多模态风险评分：加权、连续确认、交叉验证 |
| state_machine | 系统状态转换和状态动作触发 |
| actuator_manager | 蜂鸣器、RGB、风扇、继电器、水泵控制 |
| fault_manager | 传感器故障、ESP 离线、网络失败等故障标志管理 |
| comm_esp32cam | SNAP_REQ、PING/PONG、ACK、重传、错误码、结果接收 |
| log_manager | 事件计数、串口日志、可选 RAM/Flash 日志 |
| ui_manager | OLED 页面管理和状态刷新 |
| key_manager | 4 位按键扫描、消抖、短按/长按 |
| config_manager | 阈值、模式、延时等配置 |

### P0 已落地模块

当前 P0 bring-up 已在 CubeMX 工程 `firmware/stm32/SafetyMonitor` 中新增 OLED 相关文件：

| 文件 | 用途 |
|---|---|
| `Core/Inc/bsp_oled.h` | SSD1306 OLED BSP 驱动接口 |
| `Core/Src/bsp_oled.c` | SSD1306 OLED BSP 驱动实现；负责初始化、清屏、写字符、写字符串、刷新显示 |
| `Core/Inc/app_display.h` | P0 调试显示页面接口 |
| `Core/Src/app_display.c` | P0 调试显示页面实现；负责显示 KEY ADC、KEY ID、FLAME、PIR、MQ2、STATE |

`bsp_oled` 使用 `I2C1`，`PB6=SCL`，`PB7=SDA`，OLED 地址 `0x3C`。`app_display` 只负责显示上层传入的调试数据，不直接读取传感器，不直接控制蜂鸣器、RGB、继电器、风扇、水泵等执行器。

当前 P0 OLED 调试页面大致为：

```text
Safety Monitor
P0 Debug Page

KEY ADC:xxxx
KEY ID :NONE/K1/K2/K3/K4
FLAME  :0/1
PIR:0 MQ2:0
STATE  :IDLE/PIR/SMOKE/ALARM
```

当前 P0 仍处于本地硬件 bring-up 与本地安全闭环前置验证阶段：`risk_score` 风险评分和 `safety_fsm` 状态机尚未开始；OLED 已提前到 P0-04A 并验证通过。

## 4.5 任务周期建议

| 任务 | 周期 | 说明 |
|---|---:|---|
| 按键扫描 | 10 ms | 消抖和按键事件 |
| 蜂鸣器/RGB 刷新 | 20-50 ms | 闪烁和鸣叫模式 |
| 门磁/PIR/火焰 | 50-100 ms | GPIO 输入 |
| MQ-2 ADC | 100 ms | 滑动均值 |
| 风险评分 | 100-200 ms | 融合传感器状态 |
| 状态机 | 100 ms | 状态切换 |
| OLED 刷新 | 200-500 ms | 避免刷屏过快 |
| DHT 温湿度 | 1-2 s | DHT 不宜高频读取 |
| MPU6050 | 20 ms | 50 Hz 采样 |
| ESP 心跳 | 2 s | PING/PONG |
| 日志处理 | 事件触发或 1 s | 串口输出和事件计数 |

## 4.8 主循环伪代码

```c
int main(void)
{
    HAL_Init();
    SystemClock_Config();
    BSP_Init();
    Driver_Init();
    App_Init();

    while (1)
    {
        uint32_t now = HAL_GetTick();

        if (TaskDue(TASK_KEY, now))       KeyManager_Update();
        if (TaskDue(TASK_SENSOR, now))    SensorManager_Update();
        if (TaskDue(TASK_RISK, now))      RiskEngine_Update();
        if (TaskDue(TASK_STATE, now))     StateMachine_Update();
        if (TaskDue(TASK_ACT, now))       ActuatorManager_Update();
        if (TaskDue(TASK_UI, now))        UIManager_Update();
        if (TaskDue(TASK_COMM, now))      CommEsp32Cam_Process();
        if (TaskDue(TASK_LOG, now))       LogManager_Process();
    }
}
```

## 4.9 传感器管理器伪代码

```c
void SensorManager_Update(void)
{
    sensor.smoke_adc = MQ2_ReadFiltered();
    sensor.smoke_warn = sensor.smoke_adc > SMOKE_WARN_TH;
    sensor.smoke_alarm = ConfirmByCount(sensor.smoke_adc > SMOKE_ALARM_TH, &smoke_cnt, 3);

    sensor.flame_alarm = ConfirmByCount(Flame_Read() == 1, &flame_cnt, 2);
    sensor.pir_trigger = PIR_Read();
    sensor.door_open = Door_ReadOpenAfterDebounce();

    if (DHT_TimeToRead()) {
        if (DHT_Read(&sensor.temp_c, &sensor.humi_pct) != OK)
            fault.dht_fault = 1;
    }

    if (MPU_Enable()) {
        MPU_Process();
        sensor.vib_rms_g = MPU_GetRms();
        sensor.vib_pp_g  = MPU_GetPeakToPeak();
        sensor.vib_alarm = MPU_IsAlarm();
    }
}
```

## 4.10 执行器管理器伪代码

```c
void ActuatorManager_Update(void)
{
    switch (sys_state)
    {
    case SYS_NORMAL:
        SetRgb(RGB_GREEN); SetBuzzer(BUZZER_OFF);
        SetFan(0); SetRelayCutoff(0); SetPump(0);
        break;
    case SYS_PRE_ALARM:
        SetRgb(RGB_YELLOW);
        SetBuzzer(silenced ? BUZZER_OFF : BUZZER_SLOW);
        if (sensor.temp_c > TEMP_HIGH_TH || sensor.smoke_warn) SetFan(1);
        break;
    case SYS_ALARM:
        SetRgb(RGB_RED);
        SetBuzzer(silenced ? BUZZER_OFF : BUZZER_FAST);
        SetFan(1);
        break;
    case SYS_DANGER:
        SetRgb(RGB_PURPLE);
        SetBuzzer(silenced ? BUZZER_OFF : BUZZER_CONTINUOUS);
        SetFan(1); SetRelayCutoff(1); Pump_RequestLimited();
        break;
    case SYS_FAULT:
        SetRgb(RGB_BLUE_FLASH); SetBuzzer(BUZZER_SLOW); SetPump(0);
        break;
    default: break;
    }
}
```

## 4.12 STM32 固件开发方式定稿

STM32 固件技术栈定稿为 **STM32CubeMX + Keil MDK-ARM + HAL 库**。

- 使用 CubeMX 生成基础工程、时钟、GPIO、ADC、I2C、USART、TIM/PWM 等初始化代码。
- 使用 Keil MDK-ARM 编译、下载和调试 STM32F103C8T6 固件。
- 业务代码放在 `App/` 和 `BSP/` 目录：`App/` 负责状态机、风险评分、任务调度和业务流程，`BSP/` 负责对 HAL 外设接口做项目级封装。
- CubeMX 生成文件中的自定义代码必须写入 `USER CODE BEGIN` / `USER CODE END` 区域。
- 代码统一使用 HAL API，不使用、不混用 STM32F10x 标准外设库。
- 不引入 FreeRTOS，继续采用裸机前后台 + SysTick 定时调度。
