## 1.2 项目背景：三次迭代

| 阶段 | 架构 | 状态 |
|---|---|---|
| 第一版 | STM32 + ESP32-CAM + 云端 AI | 历史归档（`legacy/`） |
| 第二版 | i.MX6ULL + OPi5 双板端边协同 | 历史归档（`docs/archive/2026-imx6ull-stage/`） |
| 当前主线 | OPi5 一板双进程（safetyd + AI） | 当前演示与答辩主线 |

第一版暴露了 STM32 UART 联调成本高、ESP32-CAM 端云链路不稳定等工程问题。第二版迁移到 i.MX6ULL + OPi5 双板架构，验证了 Linux 本地安全闭环的可行性。末期 i.MX6ULL 出现供电/启动异常后，团队复用已完成的安全主控逻辑，将其迁移到 OPi5 上作为独立进程，形成当前 OPi5 一板双进程主线。

## 1.3 项目定位

本项目定位为课程级嵌入式 Linux + 边缘 AI 综合系统。OPi5 同时运行本地安全闭环（`opi5_safetyd`）和 AI 推理（`opi5_ai_service`），Flask + SQLite + React edge-console 提供本地可视化。项目保留 GPIO、I2C、PWM、MOS、供电等数字系统内容，同时引入 RKNN 本地推理和 Dashboard 可视化。

## 1.4 应用场景

| 场景 | 说明 |
|---|---|
| 实验室安全巡检 | PIR、火焰/烟雾等传感器触发本地状态机 |
| 桌面级设备健康监测 | 可选 MPU6050 监测振动 |
| 局域网本地 AI 演示 | OPi5 本地检测人形、烟雾、火焰，不依赖公网 |
| 单轴巡检 | MG90 单轴 Pan 舵机带动 USB 摄像头扫描重点区域 |

## 1.5 核心创新点

### 1.5.1 OPi5 一板双进程本地安全控制

`opi5_safetyd` 管理传感器、执行器和状态机，是唯一执行器决策点。`opi5_ai_service` 独立运行，只返回 `risk_hint`，`control_allowed=false`。两个进程互不依赖；AI 离线时安全闭环仍依据本地传感器进入安全态。

### 1.5.2 本地 RKNN 边缘推理

OPi5 承担 AI 推理，当前主线为 Qwen3-VL 2B 视觉语言模型。视觉推理、自然语言解释均在本地完成，不依赖公网。

### 1.5.3 局域网全本地闭环

系统主链路在局域网内完成：OPi5 safetyd → OPi5 AI → Flask backend → React edge-console。断网或 AI 离线时，OPi5 仍保留本地传感器安全联动。

### 1.5.4 旧平台迭代迁移

STM32/ESP32-CAM → i.MX6ULL+OPi5 → OPi5 一板三次迭代不删除，作为工程迭代证据，展示平台选择和架构重构能力。

## 1.8 为什么仍是数字系统项目

| 数字系统评分点 | 本项目体现 |
|---|---|
| 数字输入 | PIR/火焰/MQ-2 DO GPIO |
| 时序输出 | Pan 舵机 PWM、MOS 开关 |
| 总线 | I2C OLED / MPU6050 |
| 调试验证 | 逻辑分析仪记录 GPIO/I2C/PWM |
| 嵌入式系统 | OPi5 Linux、gcc 编译、systemd 服务 |
| 系统集成 | 一板双进程、状态机、Dashboard、AI 推理 |

## 1.9 为什么 AI 不控制执行器

AI 的职责是提供 `risk_hint` 和解释，它可能受到光照、遮挡、模型精度影响。危险执行器必须由 OPi5 本地 `opi5_safetyd` 根据传感器和安全规则统一决策。`control_allowed=false` 硬编码。这样既保留本地安全闭环，也避免"AI 误识别直接启动水泵"的风险。

## 1.10 一句话项目介绍

本项目使用 Orange Pi 5 同时运行本地安全闭环和 Qwen3-VL AI 推理，通过 Flask + React edge-console 展示事件和 AI 解释，实现一个低压安全、局域网全本地的 AI 多模态安全巡检系统。

## 1.11 答辩亮点表述

- "不是把单片机项目改成纯 AI，而是把单片机裸机升级为嵌入式 Linux 数字系统。"
- "AI 只给风险提示，执行器由本地 safetyd 控制，`control_allowed=false`。"
- "三次迭代（STM32 → i.MX → OPi5）体现完整工程过程和平台迁移能力。"
- "底层有 GPIO/I2C/PWM/MOS 与逻辑分析仪证据，上层有 RKNN 和 Dashboard。"
