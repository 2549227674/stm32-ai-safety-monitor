# 01 项目立项与总体设计（端边协同版）

## 1.1 项目名称体系

| 场景 | 名称 |
|---|---|
| 报告主标题 | 基于 i.MX6ULL 与 Orange Pi 5 的端边协同本地 AI 多模态安全巡检系统 |
| 英文短名 | Edge AI Safety Monitor |
| 中文短名 | 端边协同本地 AI 安全巡检系统 |
| 长期仓库名建议 | `edge-ai-safety-monitor` |
| 当前仓库处理 | `stm32-ai-safety-monitor` 新建迁移分支并归档 legacy |

## 1.2 项目背景：从端云到端边

第一版项目以 STM32 为唯一主控，ESP32-CAM 负责联网抓拍，云端 AI 负责解释。实际调试暴露出两个工程问题：第一，单片机侧硬件连线、烧录、串口桥接联调成本高；第二，端云链路依赖 ESP32-CAM 和公网/外部服务，和本地 AI 演示目标不完全一致。因此当前版本迁移到 i.MX6ULL Pro + Orange Pi 5，实现可 SSH 部署、可本地推理、可保留底层数字系统验证的端边协同方案。

## 1.3 项目定位

本项目定位为课程级嵌入式 Linux + 边缘 AI 综合系统。它既保留 GPIO、I2C、PWM、MOS、供电、逻辑分析仪等数字系统内容，又引入 V4L2 摄像头、RKNN 本地推理和 Dashboard 可视化，形成从底层硬件到本地 AI 的完整链路。

## 1.4 应用场景

| 场景 | 说明 |
|---|---|
| 实验室安全巡检 | 门磁、PIR、火焰/烟雾等传感器触发本地状态机，并抓拍复核 |
| 桌面级设备健康监测 | 可选 MPU6050 监测风扇/泵等小型设备振动 |
| 局域网本地 AI 演示 | OPi5 本地检测人形、烟雾、火焰或人脸，不依赖公网 |
| 云台巡检 | MG90 二维云台带动 USB 摄像头扫描重点区域 |

## 1.5 核心创新点

### 1.5.1 嵌入式 Linux 本地安全控制

i.MX6ULL 通过用户态控制进程 `imx_safetyd` 管理传感器、执行器和状态机。相比 STM32 裸机，Linux + SSH + systemd 的调试效率更高；相比纯上位机应用，仍然有真实 GPIO/I2C/PWM/MOS 硬件闭环。

### 1.5.2 本地 RKNN 边缘推理

Orange Pi 5 承担第四层 AI。视觉推理、可选人脸识别和自然语言解释均优先本地完成，避免把公网云作为 MVP 必需依赖。

### 1.5.3 云台巡检视觉增强

USB 摄像头固定在 MG90 二维云台上，i.MX6ULL 控制 PCA9685 输出舵机 PWM，实现低成本巡检扫描，强化“安全巡检”叙事。

### 1.5.4 局域网全本地闭环

系统主链路在局域网内完成：i.MX6ULL → OPi5 → Flask Dashboard。断网或 OPi5 离线时，i.MX6ULL 仍保留本地传感器安全联动。

### 1.5.5 旧平台迭代迁移

STM32/ESP32-CAM 阶段不删除，而是作为工程迭代证据，展示从单片机端云方案到嵌入式 Linux 端边方案的工程决策能力。

```text

```

## 1.8 为什么仍是数字系统项目

| 数字系统评分点 | 本项目体现 |
|---|---|
| 数字输入 | 门磁/PIR/火焰 DO/按键 GPIO |
| 时序输出 | PCA9685 舵机 PWM、MOS 开关 |
| 总线 | I2C 控制 PCA9685/可选 MPU6050 |
| 调试验证 | 逻辑分析仪记录 GPIO/I2C/PWM |
| 嵌入式系统 | WSL 交叉编译、Linux target、systemd 服务 |
| 系统集成 | 端边通信、状态机、Dashboard、AI 推理 |

## 1.9 为什么 AI 不控制执行器

AI 的职责是提供 `risk_hint` 和解释，它可能受到光照、遮挡、模型精度影响。危险执行器必须由 i.MX6ULL 本地状态机根据传感器和安全规则统一决策。这样既保留本地安全闭环，也避免“AI 误识别直接启动水泵/继电器”的风险。

## 1.10 一句话项目介绍

本项目使用 i.MX6ULL 进行本地安全控制与摄像头巡检，使用 Orange Pi 5 进行本地 RKNN 视觉推理，并通过 Flask Dashboard 展示事件、图片和 AI 解释，实现一个低压安全、局域网全本地的端边协同安全巡检系统。

## 1.11 答辩亮点表述

- “不是把单片机项目改成纯 AI，而是把单片机裸机升级为嵌入式 Linux 数字系统。”
- “AI 只给风险提示，执行器由本地状态机控制。”
- “旧 STM32/ESP32-CAM 是第一版工程探索，迁移体现了真实工程迭代。”
- “底层有 GPIO/I2C/PWM/MOS 与逻辑分析仪证据，上层有 RKNN 和 Dashboard。”
