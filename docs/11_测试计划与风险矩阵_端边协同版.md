# 11 测试计划与风险矩阵（OPi5 主线版）

## 11.1 测试原则

1. 先单模块，后集成。
2. 先空载，后接负载。
3. 先 mock，后真实。
4. 未实测不写通过。
5. 每项测试都要有命令、现象、结果、证据路径。

## 11.10 风险矩阵

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| 摄像头不兼容 | 中 | 中 | mock 回退，降低分辨率 |
| AI 模型加载慢 | 中 | 低 | 持久化进程，启动预热 |
| 供电不稳 | 中 | 高 | 多路 DC-DC，舵机独立供电 |
| Dashboard schema 破坏 | 低 | 中 | `raw_json` 兼容，增量迁移 |
| email 配置失效 | 低 | 低 | mock 通知日志回退 |

## 11.11 答辩前检查清单

- [ ] Dashboard 可打开，多页面可导航

- [ ] 设备离线状态可展示
- [ ] mock/真实模式切换正常
