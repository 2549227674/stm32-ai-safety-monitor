# 05 开发计划与 MVP 验收标准

## 5.1 MVP 定义

MVP 是基础安全闭环层，完全不依赖 ESP32-CAM、网络、Web、AI。

```text
烟雾/火焰/PIR/门磁任一触发
→ STM32 本地采集 → 风险评分 → 状态机进入预警/报警/危险
→ OLED 显示状态 → 蜂鸣器/RGB LED 反馈
→ 风扇/继电器/水泵至少一个执行器动作
→ 按键可布防、撤防、静音、复位
```

## 5.3 4 位按键定义

| 按键 | 短按功能 | 长按功能 |
|---|---|---|
| K1 | 布防 / 撤防 | 进入或退出手动测试模式 |
| K2 | 静音 | 取消静音 |
| K3 | 复位当前报警 | 清除事件计数 |
| K4 | OLED 页面切换 | 阈值设置/手动触发测试 |

建议先实现短按功能，长按功能作为扩展。

---

## 5.4 RGB LED 状态定义

| 状态 | 颜色/模式 | 含义 |
|---|---|---|
| IDLE | 蓝色慢闪 | 未布防 |
| ARM_DELAY | 黄色闪烁 | 布防倒计时 |
| NORMAL | 绿色常亮 | 正常监控 |
| PRE_ALARM | 黄色常亮/慢闪 | 预警 |
| ALARM | 红色快闪 | 报警 |
| DANGER | 红紫交替 | 危险 |
| SILENCE | 红色慢闪 | 已静音但仍处于报警 |
| FAULT | 蓝/紫闪烁 | 故障或外设离线 |

---

## 5.5 蜂鸣器模式

| 风险等级 | 模式 |
|---|---|
| 正常 | 关闭 |
| 预警 | 慢响：响 100 ms、停 900 ms |
| 报警 | 快响：响 100 ms、停 200 ms |
| 危险 | 连续或急促响 |
| 静音 | 关闭蜂鸣器，OLED/RGB/执行器保护保持 |

---

## 5.6 执行器本地动作策略

| 执行器 | 推荐程度 | 原因 |
|---|---|---|
| 风扇 | 最推荐 | 安全、直观、供电风险低 |
| 继电器控制低压 LED/风扇 | 推荐 | 可演示断电保护 |
| 水泵空泵/短时喷水 | 可选 | 视觉效果强，但水电风险高，建议结构稳定后再做 |

## 5.7 OLED 页面建议

正常页面：
```text
STATE: NORMAL
RISK : 1 OK
SMOKE: 820
TEMP : 28.5C
```

报警页面：
```text
STATE: ALARM
RISK : 7
TYPE : SMOKE+DOOR
ACT  : FAN ON
```

网络/AI 状态页面：
```text
NET  : ONLINE
CAM  : OK
AI   : possible_smoke
CONF : 0.82
```

故障/降级页面：
```text
FAULT FLAGS
CAM : OFFLINE
NET : OFFLINE
AI  : UNAVAILABLE
```

## 阶段 0：最小工程验证

该阶段在 P0 第 1 天正式接入传感器和状态机之前完成，只验证 CubeMX 工程链路可用。

### CubeMX 建工程 12 步

1. 新建 STM32CubeMX 工程，选择芯片 `STM32F103C8T6`。
2. 配置系统时钟，优先使用外部晶振 HSE；无外部晶振时使用 HSI。
3. 启用 `SYS`，调试接口选择 `Serial Wire`。
4. 配置基础 LED GPIO 为推挽输出。
5. 配置按键、门磁、PIR、火焰等 GPIO 输入，必须设置 Pull-up 或 Pull-down，禁止浮空。
6. 配置 USART1 作为调试串口，开启 TX/RX。
7. 配置 I2C 接口用于 OLED 和 MPU6050。
8. 配置 ADC 通道用于 MQ-2 AO。
9. 配置 TIM/PWM 用于蜂鸣器、RGB 或风扇控制。
10. 在 Project Manager 中选择 Keil MDK-ARM 工程，配置工程名和输出目录。
11. 生成代码后，只在 `USER CODE BEGIN` / `USER CODE END` 区域添加自定义代码。
12. 用 Keil 编译、下载，先验证 LED 闪烁和串口启动日志。

### 验收标准

| 验收项 | 标准 |
|---|---|
| 编译通过 | Keil 无编译错误，关键警告已确认 |
| 下载成功 | ST-Link 能下载到 STM32F103C8T6，复位后程序运行 |
| LED 闪烁 | LED 按固定周期闪烁，可证明主循环和 HAL Tick 正常 |
| USER CODE 不丢失 | 重新用 CubeMX 生成代码后，自定义代码仍保留 |

## 5.11 MVP 不是简单传感器实验的理由

1. 不是单个传感器直接点灯，而是六类多模态传感器输入。
2. 有风险评分引擎和异常分级。
3. 有完整多级状态机。
4. 有按键交互和 OLED 多页面。
5. 有蜂鸣器、RGB、多执行器联动。
6. 有故障/静音/复位/恢复逻辑。
7. 有 UART 主从通信协议。
8. 有视觉取证和 AI 异常解释。
9. 有 Web/手机端远程可视化。
10. 能独立完成现场演示，也能展示端云协同。
