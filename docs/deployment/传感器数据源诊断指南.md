# 传感器数据源诊断指南

## 快速诊断命令

### 1. 检查 systemd 服务状态

```bash
systemctl is-active opi5-ai-qwen3vl.service opi5-safetyd.service
```

输出说明：
- `active` / `running` — 服务正常运行
- `inactive` — 服务已安装但未启动
- `failed` — 服务启动失败
- `unknown` — 服务未安装或 systemd 找不到该 unit

### 2. 检查 safetyd 状态文件

```bash
cat /opt/edge-ai-safety-monitor/run/opi5_safetyd_status.json
```

预期格式：
```json
{"sensors": {"pir": 0, "flame": 0, "mq2": 0}, "ts": "2026-06-11T..."}
```

如果文件不存在或不可读，device-agent 会使用 `fallback_mock` 数据。

### 3. 检查 device-agent 实时状态

```bash
curl -s http://127.0.0.1:8090/api/status | python3 -m json.tool
```

关注字段：
- `sensors_source.safety` — `"safetyd_status_file"` 或 `"fallback_mock"`
- `sensors_source.mpu6500` — `"mock"`（当前无真实 I²C 连接）
- `sensors_source.env` — `"mock"`（当前无真实外接 MCU）
- `warnings` — 非空时表示有数据源缺失

### 4. 检查后端缓存

```bash
curl -s http://127.0.0.1:5000/api/stream/events?device_id=edge-opi5-001 | head -1
```

## 数据来源说明

| 传感器 | 来源 | 真实时机 |
|--------|------|----------|
| PIR / 火焰 / MQ-2 | safetyd_status.json | safetyd 服务运行且写入状态文件时 |
| MPU-6500 加速度/陀螺仪 | Python MockSensors | 当前始终 mock（需 I²C 真实连接） |
| DHT11 温湿度 | Python MockSensors | 当前始终 mock（需外接 MCU） |
| CPU 温度/负载/内存/磁盘 | Linux /proc /sys | 始终真实系统指标 |

## 前端显示说明

| 显示标签 | 含义 |
|----------|------|
| "实时 · safetyd" | SSE 数据新鲜且来自 safetyd 状态文件 |
| "实时 · fallback mock" | SSE 数据新鲜但 safetyd 状态文件缺失 |
| "数据滞后 Ns" | SSE 数据超过 3 秒未更新 |
| "safetyd 未运行/状态文件缺失" | SSE 连接正常但无实时数据 |
| "mock 数据 · 未接真实 MPU-6500" | MPU 数据来自 Python 模拟 |
| "外接 MCU 预留 · mock 数据" | 温湿度来自 Python 模拟 |
| "unknown / service not found or not running" | systemd 服务状态未知 |
