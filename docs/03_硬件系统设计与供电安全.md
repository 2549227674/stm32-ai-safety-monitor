## 3.2 推荐供电拓扑

当前 P0 bring-up 阶段，供电原则已从“ST-Link 3.3V 临时供面包板”调整为更清晰的 `3.3V/5V` 分轨和公共 GND 方案。黑色面包板电源模块可通过跳帽分别输出 3.3V 或 5V；两侧正极电源轨可以分别作为 3.3V 与 5V，但不得把 3.3V 正极和 5V 正极直接相连。所有传感器、电源模块、STM32、后续 ESP32-CAM 和执行器驱动板的 GND 必须与 STM32 GND 共地。

ST-Link 3.3V 只适合临时给 STM32 和少量小功耗模块供电，用于 PC13、AD 按键、蜂鸣器、RGB 等小负载 bring-up。MQ-2、继电器、风扇、水泵、ESP32-CAM 等模块不得从 ST-Link 3.3V 取电。若 STM32 已由手机 Type-C 充电器经核心板 Type-C 接口供电，则 ST-Link 只接 `SWDIO / SWCLK / GND`，不再接 ST-Link 3.3V。

```text
稳定 5V 3A 电源
 ├─ STM32F103C8T6 核心板 5V 输入
 │    └─ 板载稳压输出 3.3V 给低功耗传感器/逻辑模块
 ├─ ESP32-CAM 5V 输入，单独支路，线尽量短
 ├─ 水泵/风扇/继电器 5V 支路，经 MOS 管/驱动板/继电器模块控制
 └─ 所有 GND 必须共地
```

12V 适配器或 12V 电池不得直接接入 STM32、OLED、PIR、MQ-2 等小信号系统。小信号 bring-up 阶段避免 12V 直连；如后续低压执行器确实需要更高电压，只允许在执行器独立支路中通过继电器/MOS 驱动模块隔离控制，最终课堂演示建议低压化。

## 3.3 供电安全表

| 模块 | 推荐电源 | 注意事项 |
|---|---|---|
| STM32F103C8T6 | Type-C/5V 输入或 3.3V 稳压 | P0 小负载可临时用 ST-Link 3.3V；若核心板已由 Type-C 供电，ST-Link 只接 SWDIO/SWCLK/GND |
| ESP32-CAM | 5V 独立支路 | Wi-Fi 和拍照时峰值电流大，建议 5V 3A 总电源、短线、旁路电容 |
| MQ-2 | 多数模块 5V | DO/AO 进 STM32 前必须确认电平；5V 供电时 DO/AO 均建议分压，AO 进入 ADC 前必须不超过 3.3V |
| OLED | 3.3V/5V 模块 | I2C 上拉电平要与 STM32 兼容 |
| PIR HC-SR501 | 5V 模块 | 当前 OUT 接 PB15 已验证；与 STM32 共地 |
| 火焰传感器 | 3.3V/5V 模块 | 当前 3 针 VCC/GND/DO 模块 DO 接 PB12，无火焰状态显示正常，真实触发待安全补测 |
| GY-521 MPU6050 | 3.3V/5V 模块 | 与 OLED 共 I2C，注意地址和上拉 |
| 继电器 | 5V 模块 | IN 脚需确认 3.3V 能触发，线圈需带续流保护 |
| 风扇/水泵 | 5V 支路 | 不可由 GPIO 直接供电，必须经过驱动板或 MOS 管 |
| 蜂鸣器模块 | 3.3V 小负载 | 当前实测为低电平触发，PB5 初始电平建议 High，避免上电误响 |
| RGB KY-016 | 3.3V GPIO | P0 已验证 PB0/PB1/PB10 可控制 R/G/B；确认共阴/共阳和限流电阻 |

## 3.4 共地要求

STM32、ESP32-CAM、传感器、电机驱动板、继电器模块、5V 电源必须共地。否则会出现 UART 通信乱码、ESP32-CAM 心跳失败、传感器电平不稳定、水泵/继电器误动作、STM32 复位等问题。

建议所有 GND 回到同一电源负极，电机支路的电流回路尽量不要穿过 STM32 小信号区域。

## 3.5 防止水泵/继电器导致 STM32 复位

| 风险 | 预防措施 |
|---|---|
| 水泵启动电流拉低 5V | 使用 5V 3A 电源，水泵独立支路，线缆加粗，靠近水泵加 470uF-1000uF 电解电容 |
| 电机反电动势 | 使用带续流二极管/TVS 的驱动板，或在电机两端加续流保护 |
| 继电器线圈干扰 | 使用带光耦/续流二极管模块，继电器和 STM32 分支供电 |
| 地弹噪声 | 星形接地或粗地线，电机地线不要和 I2C/UART 细线缠绕 |
| 水泵空转 | 软件限时运行，每次喷淋 1-3 秒，冷却 10 秒以上 |
| 水路漏水 | 托盘、水管扎紧、喷头固定朝托盘，主控板抬高并远离水路 |

## 3.6 MQ-2 DO/AO 电平安全

MQ-2 模块常用 5V 供电，其 DO 和 AO 输出都可能超过 STM32 GPIO/ADC 的 3.3V 上限。当前 P0 仅完成 `PB14 / MQ2_DO` 软件链路模拟验证，MQ-2 实物 DO/AO 因缺少 10k/20k 分压电阻暂缓，不得写成实物已通过。

后续建议 DO 和 AO 都保留：DO 做快速数字报警，AO 做 ADC 数值显示和 `risk_score` 输入。MQ-2 AO 后续规划接 `PA1 / ADC1_IN1`，但 5V 供电时必须先分压再接入 STM32。处理流程：上电后用万用表测 DO/AO 最大电压；若可能超过 3.3V，必须加分压。推荐分压：上拉电阻 10k、下拉电阻 20k 左右，使 5V 约变为 3.3V。软件中 ADC 只作为趋势和阈值判断，不作为精密浓度测量。

```text
MQ-2 AO ── R1=10k ──┬── STM32 ADC
                     │
                   R2=20k
                     │
                    GND
```

N-MOSFET 模块留到 P0-05 执行器负载驱动，不用于 MQ-2 分压。PCA9685 当前不是 P0 必需模块，也不用于 MQ-2 分压。

## 3.7 水路结构安全

```text
水箱/储水容器 → 5V 小水泵 → 水管 → 喷头 → 托盘
                                     ↓
                                只喷向托盘

STM32、ESP32-CAM、电源模块必须远离水路并抬高固定。
```

前期可先空泵或 LED 模拟，不马上真实喷水。真实喷水只在托盘、防水、固定均可靠后进行。水泵运行时间限制 1-3 秒。答辩现场准备纸巾、绝缘胶带和备用断电方案。

## 3.9 备用引脚

## 3.10 引脚冲突和禁用注意

| 引脚/资源 | 注意事项 |
|---|---|
| PA13/PA14 | SWD 调试接口，不占用 |
| NRST | 复位脚，不占用 |
| BOOT0 | 启动模式脚，不接传感器 |
| PA11/PA12 | 可能用于 USB，尽量保留 |
| PA15/PB3/PB4 | 默认与 JTAG 相关，新手优先避开 |
| PC13 | 驱动能力弱，仅用于板载 LED |
| PB6/PB7 | I2C 共线，OLED 和 MPU6050 地址不能冲突 |
| PB13 | 原 PIR 方案输入链路不可靠，疑似排针/杜邦线/接触问题；当前暂不使用 |
| PB15 | 当前 PIR HC-SR501 默认引脚 |
| USART1 | 建议保留调试，不接 ESP32-CAM |

## 3.12 网络/服务器硬件说明

服务器和 Web Dashboard 不需要额外嵌入式硬件。答辩现场网络环境推荐如下：

| 组件 | 建议方案 |
|---|---|
| Wi-Fi 热点 | 手机热点或便携路由器，避免依赖教室公共网络 |
| 服务器 | 笔记本电脑运行 Flask/FastAPI，局域网 IP |
| 手机端 | 同一 Wi-Fi 下手机浏览器访问服务器 IP |
| 云端 AI | 笔记本通过手机热点 4G 访问云端 API；或使用预录结果 |

即使网络环境不佳，STM32 本地闭环仍可独立演示。Web/AI 部分可通过预录视频补充。
