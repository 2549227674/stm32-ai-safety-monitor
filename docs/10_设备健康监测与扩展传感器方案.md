# 10 设备健康监测与扩展传感器方案

## 10.1 模块定位

设备健康监测是可选扩展，不纳入 MVP。它可利用 MPU6050/GY-521 采集设备运行时的振动，用 RMS、峰峰值等简单特征判断设备状态。

## 10.2 MPU6050 是否纳入 MVP

| 结论 | 理由 |
|---|---|
| 不纳入 MVP 必须项 | 当前主线重点是 OPi5 一板安全闭环 + AI 推理 + Flask/React，MPU6050 会增加 I2C 与算法工作量 |
| 作为加分项保留 | 若 GPIO/I2C/PWM/AI 均顺利，可补充设备健康章节 |

## 10.3 Linux I2C 读取方案

MPU6050 接 OPi5 I2C5（pin3/pin5），地址 0x68。

可选实现：

- 用户态 C/C++ 使用 `/dev/i2c-*` 读取寄存器。
- Python 使用 smbus/smbus2。
- 先用 `i2cdetect` 确认设备。

## 10.4 RMS / 峰峰值算法

采样窗口：例如 1 秒 100 个样本。计算：

```text
rms = sqrt(mean(ax^2 + ay^2 + az^2))
peak_to_peak = max(norm) - min(norm)
```

阈值需基于静止、正常运行、异常振动三个场景实测。

## 10.5 与状态机结合

设备健康异常不应直接进入火灾/入侵报警，可作为 `FAULT` 或设备维护提示：

- `mpu_rms` 超过阈值：`state=FAULT` 或 `device_health=WARN`。
- 振动异常只上报 Dashboard，不控制危险执行器。

## 10.6 SoC 温度设备热健康

OPi5 芯片内置温度传感器，Linux 下直接读取，无需任何外接器件。

### 读取路径

```bash
cat /sys/class/thermal/thermal_zone0/temp   # 毫摄氏度，47000 = 47°C
```

### 定位

- `soc_temp` 是主控芯片自身温度，不是环境温度。
- 不用于火灾 ALARM 判定。
- 仅作"设备热健康"用途。

### 联动规则

`soc_temp` 高 → `device_health=WARN` → 上报 Dashboard 警告。

**不**升级为火灾 ALARM，**不**控制危险执行器。SoC 温度仅作监测上报。
