# 06 状态机与联动机制

## 6.1 系统状态定义

| 状态 | 含义 | 主要动作 |
|---|---|---|
| INIT | 初始化 | 外设初始化、自检、OLED 启动画面 |
| IDLE | 未布防 | 显示传感器值，不触发安全报警 |
| ARM_DELAY | 布防倒计时 | RGB 黄闪、蜂鸣器短响、倒计时 |
| NORMAL | 正常监控 | RGB 绿、蜂鸣器关闭、周期采集 |
| PRE_ALARM | 预警 | RGB 黄、蜂鸣器慢响、OLED 提示 |
| ALARM | 报警 | RGB 红、蜂鸣器快响、风扇/继电器动作、触发抓拍 |
| DANGER | 危险 | 红紫闪、强报警、保护动作、水泵请求 |
| SILENCE | 静音 | 蜂鸣器关闭，其他保护动作保持 |
| SNAPSHOT | 抓拍中 | 本地动作保持，等待 ESP 返回结果 |
| FAULT | 故障 | OLED 显示错误码，执行安全默认动作 |
| RECOVER | 恢复 | 异常解除后延时恢复正常 |
| MANUAL | 手动调试 | 按键测试传感器和执行器 |

CAM_OFFLINE 和 NET_OFFLINE 作为故障标志（fault_manager 管理），不完全取代 ALARM/DANGER 主状态。

## 6.2 状态说明表

| 状态 | 进入条件 | 退出条件 | 执行器动作 | OLED 显示 | 按键 |
|---|---|---|---|---|---|
| INIT | 上电 | 自检完成 | RGB 蓝/白，短响 | Booting | 不建议操作 |
| IDLE | 自检完成或撤防 | 按布防 | 执行器关闭 | Idle + 传感器值 | 布防/手动 |
| ARM_DELAY | 按布防 | 倒计时结束/撤防 | 黄闪、短响 | Arming 5..0 | 可撤防 |
| NORMAL | 布防完成且无异常 | 风险升高 | 绿灯、蜂鸣器关 | Normal | 撤防/页面 |
| PRE_ALARM | 风险 3-5 | 恢复/升级 | 黄灯、慢响、可开风扇 | Pre Alarm | 静音/复位 |
| ALARM | 风险 6-8 | 恢复/升级/静音 | 红灯、快响、风扇/继电器 | Alarm + 原因 | 静音/复位 |
| DANGER | 风险 ≥9 或火焰强触发 | 异常解除且复位 | 强报警、继电器、水泵请求 | Danger | 允许静音但保护保持 |
| SILENCE | 报警后按静音 | 新危险/复位 | 蜂鸣器关，保护保持 | Silenced | 复位/撤防 |
| SNAPSHOT | ALARM/DANGER 触发抓拍 | 成功/失败/超时 | 本地动作保持 | Snapshot | 静音 |
| FAULT | 关键故障 | 故障解除/复位 | 安全默认 | Fault Code | 复位 |
| RECOVER | 异常解除 | 延时结束 | 绿闪、蜂鸣器关 | Recover | 可撤防 |
| MANUAL | 长按模式键 | 退出手动 | 按键控制执行器 | Manual Test | 全部测试 |

## 6.3 风险评分规则

| 输入 | 条件 | 分值 |
|---|---|---:|
| 烟雾预警 | MQ-2 超过 warn 阈值 | +2 |
| 烟雾报警 | MQ-2 超过 alarm 阈值 | +3 |
| 火焰 | 火焰传感器连续触发 | +5 |
| 高温 | 温度 > 45℃ | +2 |
| 危险高温 | 温度 > 55℃ | +3 |
| PIR | 布防后人体靠近 | +1 |
| 门磁 | 布防后柜门打开 | +3 |
| 震动 | MPU6050 连续超阈值 | +3 |
| 摄像头离线 | 心跳失败 | +1（仅故障提示） |
| 多传感器 | 同时 2 个及以上异常 | +2 |
| 持续异常 | 同一异常连续超阈值 | +1 |

| 分数 | 风险等级 | 状态 |
|---:|---|---|
| 0-2 | 正常 | NORMAL |
| 3-5 | 预警 | PRE_ALARM |
| 6-8 | 报警 | ALARM |
| ≥9 | 危险 | DANGER |
| 火焰强触发 | 直接危险 | DANGER |

## 6.5 联动规则表

## 6.6 执行器触发条件

```c
// 风扇启动
fan_on = temp_high || smoke_warn || smoke_alarm || state >= SYS_ALARM;

// 继电器断电保护模拟（只控制低压负载）
relay_cutoff = (state == SYS_DANGER) || (smoke_alarm && temp_high) || manual_relay_test;

// 水泵喷淋请求
if (flame_confirmed || (risk_score >= 9 && smoke_alarm && temp_high))
    pump_request = 1;
if (pump_request && water_ok && pump_cooldown_done)
    Pump_RunLimited(1000, 3000);  // 1-3 秒

// 抓拍触发
need_snapshot = (enter_state == SYS_ALARM || enter_state == SYS_DANGER
                 || door_open_alarm || vibration_alarm);
// 抓拍成功后，ESP32-CAM 自动上传事件+图片至服务器（如在线）
// 服务器收到后自动请求云端 AI 解释（如配置）
```

## 6.7 防误报策略

| 误报来源 | 处理策略 |
|---|---|
| MQ-2 预热漂移 | 上电预热，记录 baseline，滑动均值，连续超阈值 |
| 火焰强光干扰 | 连续 2-3 次确认，固定方向，准备安全替代触发 |
| PIR 短时误触发 | 只在布防后有效，单独触发低权重 |
| 门磁抖动 | GPIO 上拉，20-50 ms 消抖 |
| DHT 失败 | 保留上次值，标记 DHT_FAULT，不直接报警 |
| MPU 偶然碰撞 | 连续多个窗口超阈值才报警 |
| ESP 离线 | 作为故障提示，不直接触发危险状态 |

## 6.8 用户按键优先级

当前 P0-02 已完成的是 AD 模拟四位按键输入验证：K1/K2/K3/K4 通过 `PA0/ADC1_IN0` 的 ADC 阈值识别。下表是后续状态机计划映射，不表示完整状态机按键业务已经完成。

| 按键 | 计划映射 | 当前状态 |
|---|---|---|
| K1 | 布防 / 撤防 | P0 已完成 ADC 识别，业务逻辑待接入 |
| K2 | 静音 | P0 已完成 ADC 识别，业务逻辑待接入 |
| K3 | 复位 | P0 已完成 ADC 识别，业务逻辑待接入 |
| K4 | 页面切换 / 手动测试 | P0 已完成 ADC 识别，业务逻辑待接入 |

P0-03 已完成蜂鸣器和 RGB 输出验证：蜂鸣器 `PB5` 低电平触发，RGB 使用 `PB0/PB1/PB10`。这些输出后续作为状态机输出通道，`NORMAL`、`PRE_ALARM`、`ALARM`、`DANGER` 等状态可分别映射不同 RGB 与蜂鸣器模式；当前 P0-03 只证明 GPIO 输出和模块电平逻辑可用，不代表完整状态机已完成。

| 优先级 | 按键事件 | 说明 |
|---:|---|---|
| 1 | K1 撤防 | 在非危险状态下允许撤防 |
| 2 | K2 静音 | 关闭蜂鸣器，但不关闭保护动作 |
| 3 | K3 复位 | 只有异常解除后才真正恢复 NORMAL |
| 4 | K4 页面/手动 | 不应影响安全状态 |
| 5 | K1 布防 | IDLE 状态进入 ARM_DELAY |

DANGER 状态下静音只关闭蜂鸣器，不关闭继电器/风扇/水泵等保护动作。

## 6.9 状态机伪代码

```c
void StateMachine_Update(void)
{
    uint8_t score = Risk_Calc(&sensor, &fault);
    RiskLevel_t level = Risk_Level(score, &sensor);

    switch (sys_state)
    {
    case SYS_INIT:
        sys_state = SelfTest_OK() ? SYS_IDLE : SYS_FAULT;
        break;
    case SYS_IDLE:
        if (Key_Event(KEY_ARM)) sys_state = SYS_ARM_DELAY;
        if (Key_Event(KEY_MANUAL_LONG)) sys_state = SYS_MANUAL;
        break;
    case SYS_ARM_DELAY:
        if (Key_Event(KEY_DISARM)) sys_state = SYS_IDLE;
        else if (ArmDelayDone()) sys_state = SYS_NORMAL;
        break;
    case SYS_NORMAL:
        if (level == RISK_PRE) sys_state = SYS_PRE_ALARM;
        else if (level == RISK_ALARM) sys_state = SYS_ALARM;
        else if (level == RISK_DANGER) sys_state = SYS_DANGER;
        break;
    case SYS_PRE_ALARM:
        if (level == RISK_NORMAL && StableFor(3000)) sys_state = SYS_NORMAL;
        else if (level == RISK_ALARM) sys_state = SYS_ALARM;
        else if (level == RISK_DANGER) sys_state = SYS_DANGER;
        if (Key_Event(KEY_SILENCE)) silenced = 1;
        break;
    case SYS_ALARM:
        if (NeedSnapshot() && Comm_CamOnline()) {
            Comm_SendSnapReq(NewEventId());
            sys_state = SYS_SNAPSHOT;
        }
        if (Key_Event(KEY_SILENCE)) sys_state = SYS_SILENCE;
        if (level == RISK_DANGER) sys_state = SYS_DANGER;
        break;
    case SYS_DANGER:
        if (NeedSnapshot() && Comm_CamOnline()) {
            Comm_SendSnapReq(NewEventId());
            sys_state = SYS_SNAPSHOT;
        }
        if (Key_Event(KEY_SILENCE)) sys_state = SYS_SILENCE;
        break;
    case SYS_SNAPSHOT:
        if (Comm_SnapDone() || Comm_SnapTimeout())
            sys_state = (level == RISK_DANGER) ? SYS_DANGER : SYS_ALARM;
        break;
    case SYS_SILENCE:
        if (level == RISK_DANGER && NewDangerEvent()) sys_state = SYS_DANGER;
        if (level == RISK_NORMAL && Key_Event(KEY_RESET)) sys_state = SYS_RECOVER;
        break;
    case SYS_RECOVER:
        if (RecoverDelayDone()) { silenced = 0; sys_state = SYS_NORMAL; }
        break;
    case SYS_FAULT:
        if (Key_Event(KEY_RESET) && Fault_CanRecover()) sys_state = SYS_IDLE;
        break;
    case SYS_MANUAL:
        Manual_Process();
        if (Key_Event(KEY_EXIT)) sys_state = SYS_IDLE;
        break;
    }
}
```
