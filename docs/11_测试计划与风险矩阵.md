# 11 测试计划与风险矩阵

## 11.1 测试原则

1. 先单模块，后集成；先本地闭环，后端云联调。
2. 先空载和低风险测试，后接水泵/继电器等外设。
3. 每完成一个阶段拍照、录屏、保存串口日志，作为报告素材。
4. 答辩前录制完整演示视频（含端云全链路）和降级演示视频。

## 11.7 风险矩阵

| 风险 | 概率 | 影响 | 预防措施 | 降级方案 |
|---|---|---|---|---|
| STM32 烧录失败 | 中 | 高 | 第一天验证 | 换板/换线/备用下载器 |
| 蜂鸣器低电平触发导致上电误响 | 中 | 中 | PB5 初始电平设为 High；代码使用 `BUZZER_ON_LEVEL=GPIO_PIN_RESET`、`BUZZER_OFF_LEVEL=GPIO_PIN_SET`；初始化后立即执行输出全关 | 先断开蜂鸣器 VCC 或信号线，只保留 RGB 调试 |
| ADC 按键半按/轻触抖动 | 中 | 中 | 使用 ADC 阈值区间、连续多次确认、短按消抖；保留 KEY_UNKNOWN 区间 | 临时用 Keil Debug Watch 观察 ADC 值，调宽阈值后再接入状态机 |
| 面包板供电线松动 | 高 | 高 | 红/蓝电源轨固定 3.3V/GND；关键线拍照记录；集成后用扎带/热熔胶固定 | 回退到单模块复测，优先恢复 STM32、按键、声光输出 |
| ST-Link 3.3V 供电能力有限 | 中 | 高 | ST-Link 供电仅限 STM32 + 小模块 P0 bring-up；ESP32-CAM、继电器、风扇、水泵使用独立电源和驱动 | 断开大负载，只用 PC13、按键、RGB/蜂鸣器做低功耗验证 |
| OLED 不显示 | 中 | 中 | 扫描 I2C 地址、检查上拉 | 暂用串口 |
| MQ-2 DO/AO 超 3.3V | 中 | 高 | 5V 供电时 DO/AO 接 STM32 前必须分压；推荐 10k/20k；AO 后续接 PA1/ADC1_IN1 | 用安全电平模拟 PB14，或暂用按键模拟烟雾 |
| PB13 输入链路不可靠 | 中 | 中 | PIR 默认改用 PB15；PB13 暂不作为传感器默认引脚 | 保留 PB13 异常记录，后续只在重新排查排针/杜邦线后再使用 |
| 12V 误接小信号系统 | 中 | 高 | 12V 适配器/电池不得直接接 STM32、OLED、PIR、MQ-2；小信号系统只使用 3.3V/5V 分轨 | 立即断电，回退单模块检查 |
| N-MOSFET/PCA9685 用错场景 | 低 | 中 | N-MOSFET 留到 P0-05 执行器驱动；PCA9685 当前 P0 暂不使用；二者都不用于 MQ-2 分压 | MQ-2 分压只用电阻网络 |
| 水泵启动导致复位 | 中 | 高 | 5V 3A、独立支路、电容 | 只演示风扇/继电器 |
| ESP32-CAM 供电不足 | 高 | 中 | 5V 独立支路、短线、电容 | 只保留本地闭环 |
| Wi-Fi 现场失败 | 高 | 中 | 手机热点、提前记录 IP | Web/AI 不演示，用视频补充 |
| 云端 AI 超时 | 中 | 中 | 设置 15 秒超时、准备模拟返回 | 使用规则化分析降级 |
| 服务器启动失败 | 低 | 中 | 提前测试、准备备份 | 用 ESP32-CAM 自身 Web |
| 接线松动 | 高 | 高 | 热熔胶、扎带 | 准备备用线 |
| 进度失控 | 中 | 高 | 四层分层开发 | 只提交本地闭环+报告+视频 |
