# 14 上报表内容与答辩摘要（端边 AI 版）

## 14.1 项目名称

基于 i.MX6ULL 与 Orange Pi 5 的端边协同本地 AI 多模态安全巡检系统

## 14.2 项目简介

本项目构建了一套基于 i.MX6ULL Pro 与 Orange Pi 5 的端边协同本地 AI 安全巡检系统。i.MX6ULL 负责传感器采集、执行器控制、V4L2 摄像头抓拍和本地状态机；Orange Pi 5 负责本地 RKNN 视觉推理；Flask + SQLite + Dashboard 负责事件存储与可视化。系统强调本地安全闭环，AI 只提供风险提示与解释，不直接控制执行器。

## 14.3 主要功能

- 多传感器本地采集与风险评分。
- Linux 本地安全状态机。
- USB 摄像头 V4L2 抓拍。
- MG90 二维云台巡检。
- Orange Pi 5 本地 AI 推理。
- Flask Dashboard 展示事件、图片、AI 摘要。
- 断网/AI 离线降级。

## 14.4 技术路线

```text
PC/WSL 开发与交叉编译
  → SSH 部署 i.MX6ULL 控制程序
  → i.MX6ULL 采集传感器与抓拍图片
  → HTTP POST 到 Orange Pi 5 AI 服务
  → i.MX6ULL 融合 AI risk_hint 与本地状态机
  → Flask /api/events 和 /api/images
  → Dashboard 展示
```

## 14.5 创新点

| 创新点 | 说明 |
|---|---|
| 端边协同 | 将云端 AI 替换为局域网本地 OPi5 推理 |
| 本地安全优先 | AI 只提示，执行器由 i.MX6ULL 状态机控制 |
| 嵌入式 Linux 数字系统 | GPIO/I2C/PWM/MOS 与 V4L2 综合应用 |
| 云台巡检 | PCA9685 + MG90 实现低成本视觉扫描 |
| 工程迁移能力 | 从 STM32/ESP32-CAM 迭代到 i.MX6ULL/OPi5 |

## 14.8 摘要模板

### 50 字

基于 i.MX6ULL 与 Orange Pi 5，实现本地安全控制、V4L2 抓拍、RKNN 推理和 Dashboard 展示的端边协同安全巡检系统。

### 100 字

本项目采用 i.MX6ULL Pro 作为本地安全控制与视觉采集节点，Orange Pi 5 作为本地 RKNN 推理节点，结合 Flask + SQLite + Dashboard 实现安全事件记录与可视化。系统通过 GPIO/I2C/PWM/MOS 完成底层控制，通过 V4L2 与本地 AI 完成视觉复核，强调 AI 不直接控制执行器。

### 300 字

本项目面向实验室与设备安全巡检场景，设计并实现一套基于 i.MX6ULL 与 Orange Pi 5 的端边协同本地 AI 多模态安全巡检系统。系统由四层组成：i.MX6ULL Linux 本地安全控制层负责传感器采集、MOS 负载控制与状态机；i.MX6ULL 视觉采集层通过 USB 摄像头和 MG90 云台完成巡检抓拍；Flask + SQLite + Dashboard 负责本地事件存储、图片和 AI 结果展示；Orange Pi 5 负责 RKNN 本地视觉推理和可选自然语言解释。项目从原 STM32 + ESP32-CAM 端云方案迁移到嵌入式 Linux + 本地边缘 AI 方案，保留 `risk_score`、状态机和事件 JSON 语义，同时提高开发调试效率。系统安全策略为本地闭环优先，AI 只返回风险提示，不直接控制执行器，并通过独立供电、共地、默认 OFF、低压负载等措施保证演示安全。
