# 14 上报表内容与答辩摘要（OPi5 主线版）

## 14.1 项目名称

基于 Orange Pi 5 的本地 AI 多模态安全巡检系统

## 14.2 项目简介

本项目构建了一套基于 Orange Pi 5 的本地 AI 安全巡检系统。OPi5 同时运行本地安全闭环（`opi5_safetyd`）和 AI 推理（`opi5_ai_service`），通过 `opi5-device-agent` 实现心跳/遥测/视频/AI observation 上报，Flask + React edge-console 负责事件存储与可视化。系统强调本地安全闭环，AI 只提供风险提示与解释，不直接控制执行器。

## 14.3 主要功能

- OPi5 GPIO 传感器本地采集与风险评分。
- OPi5 本地安全状态机（NORMAL/WARN/VERIFY/ALARM/FAULT）。
- USB 摄像头 MJPEG 视频流与快照。
- Qwen3-VL 本地 AI 推理。
- device-agent 心跳/遥测/AI observation 上报。
- Flask API + React edge-console Dashboard 可视化。
- email 通知。
- 断网/AI 离线降级。

## 14.4 技术路线

```text
OPi5 一板运行三个独立进程：
  opi5_safetyd     — GPIO 传感器 → 状态机 → 执行器
  opi5_ai_service  — Qwen3-VL 推理，control_allowed=false
  opi5-device-agent — 心跳/遥测/视频/AI observation

Flask backend ← device-agent 上报 → React edge-console Dashboard
```

## 14.5 创新点

| 创新点 | 说明 |
|---|---|
| OPi5 一板双进程 | 安全闭环与 AI 推理独立运行，互不依赖 |
| 本地安全优先 | AI 只提示，执行器由 safetyd 状态机控制 |
| 嵌入式 Linux 数字系统 | GPIO/I2C/PWM/MOS 综合应用 |
| 工程迁移能力 | 从 STM32/ESP32-CAM → i.MX6ULL+OPi5 → OPi5 一板三次迭代 |

## 14.8 摘要模板

### 50 字

基于 Orange Pi 5，实现本地安全控制、Qwen3-VL AI 推理和 Dashboard 展示的本地 AI 多模态安全巡检系统。

### 100 字

本项目采用 Orange Pi 5 同时运行本地安全闭环和 Qwen3-VL AI 推理，通过 device-agent 实现心跳/遥测/视频/AI observation 上报，Flask + React edge-console 实现事件存储与可视化。系统通过 GPIO/I2C/PWM 完成底层控制，强调 AI 不直接控制执行器，体现三次迭代的工程迁移能力。

### 300 字

本项目面向实验室与设备安全巡检场景，设计并实现一套基于 Orange Pi 5 的本地 AI 多模态安全巡检系统。系统由 OPi5 一板三进程架构组成：`opi5_safetyd` 负责 GPIO 传感器采集、执行器控制与安全状态机；`opi5_ai_service` 负责 Qwen3-VL 视觉推理，返回 risk_hint 与自然语言解释；`opi5-device-agent` 负责心跳/遥测/视频流/AI observation 上报。Flask + SQLite 后端提供事件存储、遥测查询、通知配置等 API，React edge-console 提供 Dashboard 可视化。项目从原 STM32 + ESP32-CAM 端云方案，经 i.MX6ULL + OPi5 双板端边方案，迭代到当前 OPi5 一板方案，保留 risk_score、状态机和事件 JSON 语义。系统安全策略为本地闭环优先，AI 只返回风险提示，不直接控制执行器（control_allowed=false），并通过独立供电、共地、默认 OFF、低压负载等措施保证演示安全。
