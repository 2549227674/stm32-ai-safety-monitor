## PCA9685 初始化问题排查

### 问题 1：SLEEP 模式

PCA9685 MODE1 寄存器读到 0x11（SLEEP=1），芯片处于睡眠模式，PWM 不工作。

原因：pca9685_set_pose 退出后芯片未保持唤醒状态。

修复：pca9685_set_pwm 工具在每次设置 duty 前显式写 MODE1=0x20（AI=1, SLEEP=0）。

### 问题 2：AI (Auto-Increment) 位

MODE1 写入后读到 0x01（AI=0），导致多字节 I2C 写入不正确。

原因：wake 函数保留了旧 MODE1 的其他位但丢失了 AI 位。

修复：显式设置 MODE1 = 0x20（AI=1, SLEEP=0），与 i2cset 行为一致。

### 验证

i2cset 直接写入确认芯片正常：
```bash
i2cset -y 0 0x40 0x00 0x20        # MODE1: AI=1, SLEEP=0
i2cset -y 0 0x40 0x12 0x00 0x10 0x00 0x00 i  # CH3 full on
```

修复后 C 代码读回正确：ON_H=0x10 (full on), OFF_H=0x00。

## 工具：pca9685_set_pwm

新增 `edge/imx6ull-controller/src/pca9685_set_pwm.c`：

- 支持 channel 0-15, duty 0-4095
- 自动唤醒 PCA9685（MODE1=0x20）
- 不改变 PWM 频率（保留 50Hz，不影响舵机 CH0/CH1）
- CLI: `pca9685_set_pwm --bus /dev/i2c-0 --addr 0x40 --channel N --duty N`

## 代码修改

修改 `edge/imx6ull-controller/src/imx_safetyd.c`：

1. 新增配置：
   - `RGB_BACKEND=gpio|pca9685`（默认 gpio）
   - `PCA9685_BUS`（默认 /dev/i2c-0）
   - `PCA9685_ADDR`（默认 0x40）
   - `PCA9685_RGB_R_CH/G_CH/B_CH`（默认 2/3/4）
   - `PCA9685_RGB_ON_DUTY`（默认 4095）
   - `PCA9685_RGB_OFF_DUTY`（默认 0）

2. 新增函数 `pca9685_set_rgb()`：通过子进程调用 pca9685_set_pwm

3. `apply_actuators_by_state()` 支持后端切换：
   - `rgb_backend=gpio`：沿用 Task10-E GPIO 方式
   - `rgb_backend=pca9685`：走 PCA9685 CH2/CH3/CH4

4. 蜂鸣器始终走 gpio122，不迁移到 PCA9685

5. `all_off()` 按后端关闭 RGB，蜂鸣器始终 GPIO 关闭

## imx_safetyd 验证

### 场景 A：NORMAL（PCA9685 绿灯）

```bash
RGB_BACKEND=pca9685 POST_NORMAL=1 GPIO_MQ2_ACTIVE_HIGH=0 timeout 8 ./imx_safetyd --mode loop
```

PCA9685 输出：CH3 duty=4095 (G=1), CH2/CH4 duty=0
日志：`[actuators] state=NORMAL buzzer=0 R=0 G=1 B=0 rgb_backend=pca9685`
绿灯亮 ✓

### 场景 B：VERIFY（PCA9685 蓝灯）

```bash
RGB_BACKEND=pca9685 FORCE_VERIFY=1 GPIO_MQ2_ACTIVE_HIGH=0 timeout 8 ./imx_safetyd --mode loop
```

PCA9685 输出：CH4 duty=4095 (B=1), CH2/CH3 duty=0
日志：`[actuators] state=VERIFY buzzer=0 R=0 G=0 B=1 rgb_backend=pca9685`
蓝灯亮 ✓

### 场景 C：ALARM（PCA9685 红灯 + GPIO 蜂鸣器）

```bash
RGB_BACKEND=pca9685 GPIO_MQ2_ACTIVE_HIGH=1 timeout 8 ./imx_safetyd --mode loop
```

PCA9685 输出：CH2 duty=4095 (R=1), CH3/CH4 duty=0
GPIO：gpio122=0（buzzer on, active low）
日志：`[actuators] state=ALARM buzzer=1 R=1 G=0 B=0 rgb_backend=pca9685`
红灯亮 ✓，蜂鸣器响 ✓

## 结论

**Task10-E2 通过**：
- PCA9685 CH2/CH3/CH4 PWM 驱动 RGB 全部正常
- 新 RGB 模块三路均亮，亮度良好
- RGB_BACKEND 可选 gpio/pca9685
- 蜂鸣器仍走 gpio122 active low
- 本地安全报警核心（蜂鸣器）不依赖 PCA9685
- RGB PCA9685 是视觉增强，不承担安全报警核心

## 边界

- 蜂鸣器仍 gpio122，不经 PCA9685
- CH0/CH1 舵机保持不动，PCA9685 频率 50Hz 未改
- 不接继电器/风扇/水泵/MOS/220V
- AI/OPi5/Flask 不控制执行器
- `control_allowed` 保持 `false`
- Task10-E GPIO RGB 证据保留，Task10-E2 是增强方案
