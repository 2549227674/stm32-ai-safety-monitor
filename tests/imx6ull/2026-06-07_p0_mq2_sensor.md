## 裸读测试

### 第一轮：电位器调节触发

用户通过调节模块电位器观察 DO 翻转。100 个采样（200ms 间隔）：

| 时间 | mq2_raw | 说明 |
|---|---|---|
| 0-23.8s | 全 1 | 触发状态（电位器阈值低） |
| 23.8-30s | 全 0 | 未触发（用户调高阈值） |

结论：**mq2_active_high = 1**（raw=1 触发，raw=0 正常），5V + DO 直连课设方案可行。

### 注意事项

- gpio120 被设备树 ecspi1 占用，需先 unbind ecspi1 驱动才能导出 GPIO
- `echo "2008000.ecspi" > /sys/bus/platform/drivers/spi_imx/unbind`
- MQ-2 传感器拔掉后 gpio120 悬空读 1（无内部下拉）
- 通过 GPIO_MQ2_ACTIVE_HIGH 反转逻辑完成代码验证

## 代码修改

修改 `edge/imx6ull-controller/src/imx_safetyd.c`：

1. 新增 `GPIO_MQ2`（默认 120）和 `GPIO_MQ2_ACTIVE_HIGH`（默认 1）配置
2. `read_gpio()` 新增 MQ-2 GPIO 读取，失败时 mq2=0 且记录 diagnostics
3. `compute_state()` 新增强触发规则：`mq2=1 → ALARM`，risk_score += 5
4. flame=1 和 mq2=1 可叠加，risk_score clamp 到 10
5. `build_event_json()` diagnostics 新增 mq2_gpio/mq2_raw/mq2_active_high
6. `write_status_json()` 新增 last_mq2 字段

## imx_safetyd once 验证

### 场景 A：无烟（active_high=0，raw=1→mq2=0）

```
GPIO_MQ2=120 GPIO_MQ2_ACTIVE_HIGH=0 ./imx_safetyd --mode once
```

结果：
- `mq2_raw=1, mq2=0`（反转逻辑）
- `state=NORMAL, risk_score=0`
- `status.json: last_mq2=0, last_state="NORMAL", last_risk_score=0`

### 场景 B：有烟（active_high=1，raw=1→mq2=1→ALARM）

```
GPIO_MQ2=120 GPIO_MQ2_ACTIVE_HIGH=1 ./imx_safetyd --mode once
```

结果：
- `mq2_raw=1, mq2=1`
- `state=ALARM, risk_score=5`
- `need_snap=true` → 抓拍 → AI offline fallback
- `status.json: last_mq2=1, last_state="ALARM", last_risk_score=5`
