# Task11-B: Relay CH5 验证

## 1. 基本信息

| 项目 | 值 |

## 2. 接线

| 项目 | 说明 |
|---|---|
| relay 型号 | KY-019 5V 单路继电器模块 |
| VCC | 独立 5V |
| GND | GND（与 i.MX/PCA9685 共地） |
| IN | PCA9685 CH5 PWM/S 信号脚 |
| 蓝色端子 | NC/COM/NO 本轮未接负载 |

## 3. 默认 OFF 结果

上电未运行程序时：继电器不吸合，指示灯灭。**通过**。

## 4. CH5 duty 裸测

| duty | 指示灯 | 结论 |
|---|---|---|
| 0 | 灭 | relay OFF |
| 4095 | 亮 | relay ON |

结论：**relay active high**（duty=4095 吸合，duty=0 释放）。

## 5. 代码修改摘要

imx_safetyd.c 新增：
- `RELAY_ENABLE=0|1` 环境变量（默认 0）
- `PCA9685_RELAY_CH=5` 环境变量（默认 5）
- `PCA9685_RELAY_ACTIVE_HIGH=1|0` 环境变量（默认 1）
- `pca9685_set_relay()` 函数：通过 pca9685_set_pwm 控制 CH5
- `apply_actuators_by_state()`：ALARM 时 relay=1，NORMAL/VERIFY/FAULT 时 relay=0
- `all_off()`：relay off
- 事件 JSON：`actuators.relay` 使用真实值
- CLI 参数：`--relay-enable`、`--pca9685-relay-ch`、`--pca9685-relay-active-high`

## 6. imx_safetyd 验证

### 场景 A：NORMAL

```bash
RELAY_ENABLE=1 RGB_BACKEND=pca9685 POST_NORMAL=1 GPIO_MQ2_ACTIVE_HIGH=0 timeout 8 ./imx_safetyd --mode loop
```

结果：
- state=NORMAL
- CH5 duty=0（relay OFF）
- 指示灯灭
- RGB 绿灯

### 场景 B：ALARM

```bash
RELAY_ENABLE=1 RGB_BACKEND=pca9685 GPIO_MQ2_ACTIVE_HIGH=1 timeout 10 ./imx_safetyd --mode loop --interval-sec 3
```

结果：
- state=ALARM（mq2=1 触发）
- CH5 duty=4095（relay ON）
- 指示灯亮
- 蜂鸣器响、RGB 红灯
- relay=1 事件字段正确

### 退出 all_off

程序结束后：
- CH5 duty=0（relay OFF）
- 指示灯灭
- 蜂鸣器停

## 7. 结论

**通过**：Task11-B relay CH5 通过

- 默认 OFF：上电不吸合
- PCA9685 CH5 active high：duty=4095 吸合，duty=0 释放
- ALARM 时 relay 动作，NORMAL/VERIFY 时 relay OFF
- FAULT 时 relay OFF（避免误动作）
- all_off 确保退出时 relay 释放
- relay 写失败不导致主控崩溃
