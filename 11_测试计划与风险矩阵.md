# 11 测试计划与风险矩阵

## 11.1 测试原则

1. 先单模块，后集成；先本地闭环，后端云联调。
2. 先空载和低风险测试，后接水泵/继电器等外设。
3. 每完成一个阶段拍照、录屏、保存串口日志，作为报告素材。
4. 答辩前录制完整演示视频（含端云全链路）和降级演示视频。

## 11.4 ESP32-CAM 与通信测试

| 测试项 | 方法 | 预期结果 |
|---|---|---|
| PING/PONG | STM32 周期 PING | OLED 显示 CAM_OK |
| SNAP_REQ | 触发 ALARM | ESP 返回 ACK |
| 抓拍成功 | ESP 拍照成功 | STM32 显示 SNAP_OK |
| 抓拍失败 | 模拟摄像头失败 | SNAP_FAIL，本地继续 |
| ACK 超时 | 断开 UART | 重传 3 次后 CAM_OFFLINE |
| 拔掉 ESP | 报警前拔电源 | CAM_OFFLINE，本地报警正常 |

## 11.5 端云联调测试

| 测试项 | 方法 | 通过标准 |
|---|---|---|
| Web Dashboard | 手机/电脑访问局域网 IP | 能看到状态、传感器值、抓拍图片 |
| 事件上传 | 触发报警后检查服务器 | 服务器收到事件 JSON |

| 历史查询 | Web 页面查看历史事件 | 能看到最近 5 条事件 |

| AI 展示 | Web 页面查看 AI 结果 | 显示异常类别、描述、建议 |

| 手机端 | 手机浏览器访问 | 页面响应式显示正常 |
| 断网 | 关闭 Wi-Fi | Web/AI 失败，本地报警保持 |
| 全链路 | 异常→报警→抓拍→上传→AI→Web | 3-5 分钟完整流程 |

## 11.7 风险矩阵

| 风险 | 概率 | 影响 | 预防措施 | 降级方案 |
|---|---|---|---|---|
| STM32 烧录失败 | 中 | 高 | 第一天验证 | 换板/换线/备用下载器 |
| OLED 不显示 | 中 | 中 | 扫描 I2C 地址、检查上拉 | 暂用串口 |
| MQ-2 AO 超 3.3V | 中 | 高 | 万用表测量，必要时分压 | 用 DO 或按键模拟 |
| 水泵启动导致复位 | 中 | 高 | 5V 3A、独立支路、电容 | 只演示风扇/继电器 |
| ESP32-CAM 供电不足 | 高 | 中 | 5V 独立支路、短线、电容 | 只保留本地闭环 |
| Wi-Fi 现场失败 | 高 | 中 | 手机热点、提前记录 IP | Web/AI 不演示，用视频补充 |
| 云端 AI 超时 | 中 | 中 | 设置 15 秒超时、准备模拟返回 | 使用规则化分析降级 |
| 服务器启动失败 | 低 | 中 | 提前测试、准备备份 | 用 ESP32-CAM 自身 Web |
| 接线松动 | 高 | 高 | 热熔胶、扎带 | 准备备用线 |
| 进度失控 | 中 | 高 | 四层分层开发 | 只提交本地闭环+报告+视频 |
